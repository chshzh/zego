## 6. Fixes Applied During This Verification

| File | Fix | Severity |
|------|-----|----------|
| `bricks/network/src/wifi_utils.c` | `p2p_gc_start_pairing_internal()` was calling `net_mgmt(NET_REQUEST_WIFI_DISCONNECT)` directly in the button `ZBUS_LISTENER` context. Moved the disconnect into `p2p_gc_do_connect()` (runs on the dedicated 4 KB P2P work queue); the listener now only sets state + schedules work. Matches the codebase rule that P2P `net_mgmt` calls must not run on the 2 KB listener/sysworkq stack. Rebuilt clean. | P1 (fixed) |
